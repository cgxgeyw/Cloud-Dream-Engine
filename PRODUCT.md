# Product

## Register

product

## Users

World authors build local, model-driven narrative games. Players run those worlds on desktop and Android, often using a world-specific interface for long sessions.

## Product Purpose

Cloud Dream Engine lets authors create, package, and play portable narrative worlds while keeping game state and model access under the local application. A world owns the presentation of its game page, while the application owns trusted capabilities and persistence.

## Brand Personality

Focused, expressive, and author-friendly. The editor should stay predictable and quiet; each world may establish its own visual identity inside the game surface.

## Anti-references

Avoid a generic SaaS dashboard, decorative card grids, shell styling that leaks into a world, and mobile layouts that are desktop screens merely scaled down.

## Design Principles

1. Keep app-shell controls familiar and task-focused.
2. Give world authors broad visual control inside an explicit runtime boundary.
3. Treat desktop and mobile as separate complete layouts.
4. Make capability ownership and failure states visible and deterministic.
5. Preserve existing world data through versioned migration paths.

## Accessibility & Inclusion

Support keyboard navigation, visible focus states, semantic controls, reduced motion, readable contrast, text resizing, and mobile safe areas. World styling may vary, but trusted runtime controls must remain operable.
